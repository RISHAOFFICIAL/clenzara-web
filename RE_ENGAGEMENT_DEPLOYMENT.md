## Step 3: Deploy the Re-Engagement Backend

1. In your Google Sheet → **Extensions → Apps Script**
2. Open the existing script (from the waitlist setup)
3. **Copy the entire contents of `/home/team/shared/re-engagement-backend.gs`** and append it to your existing script — OR create a new script file and paste it there
4. **⚠️ REQUIRED: Update the CONFIG object:**
   - `OWNER_EMAIL` → Your actual email
   - `PHYSICAL_ADDRESS` → Your business address (CAN-SPAM requirement)
   - `SENDER_NAME` → "The Clenzara Team" (or your name)
5. Click **Save** (Ctrl+S)

5. Verify:

## Troubleshooting

### "Daily send limit exceeded"
- Free Gmail: ~100 recipients/day
- Google Workspace: ~1,500 recipients/day
- Solution: Reduce BATCH_SIZE in CONFIG to 20-30

### "Script function not found"
- Make sure you saved the script after pasting
- Check that no syntax errors exist (Apps Script editor highlights issues)

### MailApp not working
- You MUST run `manualTestAndAuthorize()` first to grant permissions
- If still failing, go to Apps Script → Executions to see error details

### Imported CSV has wrong columns
- The extract script outputs: Email, EventName
- Headers must match exactly for the send logic to work
